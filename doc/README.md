# Documentation Map

This folder is organized by topic and document lifecycle.

## Active docs

- `doc/auth/` - canonical authentication flow docs
- `doc/bookings/` - booking lifecycle, walk-in, outsource, and room state docs
- `doc/payments/` - payment integration and email/payment workflow docs
- `doc/staff/` - RBAC and hotel staff management docs
- `doc/media/` - image and file storage docs
- `doc/ratings/` - rating system docs
- `doc/analytics/` - analytics design and implementation docs
- `doc/notes/` - cross-cutting technical notes and references
- `doc/hotel-booking/` - stable hotel booking reference docs

## Legacy docs

- `doc/archive/` - deprecated, superseded, or misfiled notes
- `doc/auth/archive/` - older auth drafts and refactor history

## Guidance

- Keep canonical docs in the topic folder that matches the feature they describe.
- Move drafts and superseded documents into an `archive/` subfolder instead of deleting them.
- Use concise, descriptive filenames so the folder structure can stand on its own.
